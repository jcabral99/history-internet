Triple "A" Testing (AAA)



Why AAA Testing

Triple "A" Testing or AAA is important because it helps with writing good tests and structure cases for your code. The importance of the process allows for time to be saved and being able to test your work throughout the process which leads to cheaper, faster, and easier work. It makes the code a more agile process. It has been shown that AAA testing improves the quality of the code which later ensures that software bugs are found more easily and earlier on. The overall testing process gives any coder an exact root to the error without having to complete th code.

The A's stand for Arrange, Act, and Assert. The structure of the code will follow those exact words.

- Arrange: Inputs and targets; sets up the test case
- Act: Target behavior; covers the main thing to be tested; calls a function or method
- Assert: Expected outcomes; should elicit some sort of response; ultimately determines if the test passes or fails


AAA Testing Tutorial

The only way to understand what AAA Testing means is to see how it works within the coding process. Below you will find an example of arrange, act, and assert demonstrated in the coding of a calculator.

All of the code will be written in Python. The following are examples of AAA testing for addition and multiplication.

  • Testing the Add function:
  • def test_calculator_add():
  • #Arrange by instantiation the calc class
  • calc = Calculator()
  • #Act by calling the method to test
  • calc.add_number(4)
  • #Assert that the results are correct
  • assert calc.result == 4
  • Testing the Multiplication function:
  • def test_calculator_multiply():
  • #Arrange by instantiation the calc class
  • calc = Calculator()
  • #Act by calling the method to test
  • result = calc.multiply_number(1,2)
  • #Assert that the results are correct
  • assert calc.result == 2
