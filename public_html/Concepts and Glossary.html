Pylint Beginners

Every good programmer will know how to use Python, but they will also know how to apply Pylint to any code that they are writing. Pylint will allow for you to test the information that you are writing up. It will show any user what errors they are making within their code, which will help them avoid problems later on.


There are a series of steps to take to be able to implement Pylint within your code. You will begin by installing Pylint:

  • You need to add the file .pylintrc, which will allow for pylint to be configured
  • Typically, the .pylintrc file will have a makeup of what you want to be checked and what you DO NOT want to be checked for specific errors
  • You will also need to add a requirements.txt file which will include a list of what tests you want to run
    • - Some examples would be pytest-cov, pytest, pytest-pylint
  • Once those files are created you will need to run "Pip Install" within the terminal. This will make sure that all tests are configured and you will be able to test the code through commands
  • Within the terminal you will write pip install -r requirements.txt this command will configure all the tests listed out in the requirement.txt file
  • Once that is complete you are able to run the command: pytest --pylint
  • The commands will run and it will give back any errors of failed code. It will also give suggestions on how to help fix any errors within the code and will also show what test have passed within your code.

All of these steps will help you run Pylint through your code. It will help any beginner find errors within their code and ensure that they are able to fix any issues.




Object-Oriented Programming Glossary

Below you will find a glossary of terms that will help you understand some basics of Object-Oriented Programming (OOP). If you navigate to the Python dropdown menu in the OOP tab you will be able to learn more about how different concepts are applied within a program.

  • Object – An instance of a class that is created at runtime. In C#, an instance is created with the new keyword
  • Class – Code describing a specific entity in the application. Class contains the code defining the properties and methods
    • - Example: Student or address
  • Property – Code defining the data managed by the class
    • - Example: StudentName or EmailAddress
  • Method – code defining the actions or behaviors of the class. Methods are defined in the code with C# functions
    • - Example: Validate or CalculateTotal
  • Constructor – Code that is executed each time an instance of the class is created
  • Unit test – Ensure that each individual part of the code is working well and that it is supposed to work
  • Object-Oriented Programming – A design and building application that is easy, natural, well- crafted, and testable by focusing on objects that interact cleanly with one another
  • Static - Does not receive an implicit first argument. This is bound to the class and not the object of the class.
  • Class Method - It is a built-in function decorator that is an expression that gets evaluated after your function is defined. The results shadows your function definition
